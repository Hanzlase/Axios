## вҡЎ What is Axios?

**Axios** is a local-first RAG (Retrieval-Augmented Generation) workspace that turns your raw documents into structured, actionable knowledge вҖ” instantly.

Upload any document. Axios retrieves the right evidence using **hybrid search**, then generates precisely what you need: detailed explanations, interactive quizzes, spaced-repetition flashcards, or complete study plans вҖ” all grounded in *your* content, not hallucinations.

## рҹҡҖ Features

| | Feature | Description |
|---|---|---|
| рҹ”Қ | **Hybrid Retrieval** | Vector search + BM25 keyword matching + neural reranking for maximum recall precision |
| рҹ§  | **Structured Modes** | Auto-routed generation: `chat`, `explain`, `quiz`, `flashcards`, `plan` |
| рҹ“Һ | **Grounded Citations** | Every response ships with source references вҖ” no black-box answers |
| вҡЎ | **Streaming UX** | Real-time SSE token streaming вҖ” output appears as it's generated |
| рҹ”’ | **Local-First** | Session-scoped document context вҖ” your documents never leave your session |
| рҹҢҗ | **Production-Ready** | Railway-deployable, Docker-supported, CORS-configured two-service architecture |

## вҡҷпёҸ Quickstart

### Prerequisites

- **Node.js** 18+
- **Python** 3.10+

### 1 вҖ” Clone

```bash
git clone https://github.com/Hanzlase/Axios.git
cd axios
```

### 2 вҖ” Backend (FastAPI)

```bash
cd backend
pip install -r requirements.txt
python main.py
```

> API live at вҶ’ `http://localhost:8000`  
> Swagger docs вҶ’ `http://localhost:8000/docs`

### 3 вҖ” Frontend (Next.js)

```bash
cd frontend
npm install
npm run dev
```

> App live at вҶ’ `http://localhost:3000`

### 4 вҖ” Environment

Create `backend/.env`:

```env
OPENROUTER_API_KEY=your_key_here
COHERE_API_KEY=your_key_here
ALLOW_ORIGINS=http://localhost:3000
LOG_LEVEL=INFO
```

## рҹҢҚ Deploy on Railway

Axios deploys as **two independent Railway services** вҖ” backend and frontend вҖ” in one Railway project.

```
Railway Project
в”ңв”Җв”Җ backend-service   (FastAPI)   вҶҗ Root: /backend
в””в”Җв”Җ frontend-service  (Next.js)   вҶҗ Root: /frontend
```

### Step 1 вҖ” Deploy Backend

1. Railway вҶ’ **New Project** вҶ’ **Deploy from GitHub**
2. Set Root Directory: `backend`
3. Railway starts the API via `backend/Procfile` on `$PORT`

**Backend environment variables:**

```env
APP_ENV=production
LOG_LEVEL=INFO
OPENROUTER_API_KEY=<your_key>
COHERE_API_KEY=<your_key>
CORS_ALLOW_ORIGINS=<frontend_url>       # Set after frontend deploy
FRONTEND_ORIGIN=<frontend_url>          # Set after frontend deploy
```

Verify deployment:
- `https://<backend>.up.railway.app/health`
- `https://<backend>.up.railway.app/docs`

### Step 2 вҖ” Deploy Frontend

1. In the same project вҶ’ **New Service** вҶ’ GitHub repo
2. Set Root Directory: `frontend`
3. Builds via `frontend/Dockerfile`

**Frontend environment variables:**

```env
NEXT_PUBLIC_API_BASE_URL=https://<backend>.up.railway.app
```

### Step 3 вҖ” Wire CORS

After frontend deploys, copy its public URL and update the **backend** service:

```env
CORS_ALLOW_ORIGINS=https://<frontend>.up.railway.app
FRONTEND_ORIGIN=https://<frontend>.up.railway.app
```

Trigger a backend redeploy. You're live. рҹҺү

## рҹ“Ў API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/upload` | Upload session documents for ingestion |
| `GET` | `/api/uploads/{session_id}` | List session files and processing status |
| `POST` | `/api/chat/stream` | SSE stream вҖ” conversational chat mode |
| `POST` | `/api/agent/stream` | SSE stream вҖ” structured agent modes (quiz, explain, flashcards, plan) |
| `GET` | `/health` | Liveness probe |

## рҹ”§ Configuration Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENROUTER_API_KEY` | Optional | Primary LLM provider (OpenAI-compatible) |
| `COHERE_API_KEY` | Optional | Fallback LLM + reranking provider |
| `ALLOW_ORIGINS` | Yes (prod) | CORS allowlist (comma-separated URLs) |
| `FRONTEND_ORIGIN` | Yes (prod) | Trusted frontend origin |
| `APP_ENV` | Optional | `development` or `production` |
| `LOG_LEVEL` | Optional | Log verbosity вҖ” `DEBUG`, `INFO`, `WARNING` |
